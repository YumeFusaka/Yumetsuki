# Yumetsuki UI 规范

## 设计方向

Yumetsuki 的 UI 不是通用后台，也不是网页表单工具。它的风格应服务于：

- 桌宠角色感
- 粉白樱花主题
- 轻盈、柔和、现代
- 细节精致，但不过度装饰

## 颜色

- 主背景渐变：
  `#fff5f7 -> #ffebf2 -> #fae4f0`
- 强调色：
  `#d4567a`
- 深强调色：
  `#9b3060`
- 主文字：
  `#4a3040`
- 次级文字：
  `#6b4a5a`

## 通用控件风格

### 输入框

- 半透明浅白底
- 默认使用纯玫瑰色 `2px` 实边框
- 随聊天窗整体缩放按比例变化，最小不低于 `1px`
- focus 状态使用更深一档玫瑰色

### 下拉框

- 设置中心所有 `QComboBox` 必须复用 `ui.theme.SAKURA_COMBO_BOX_STYLE`。
- 右侧箭头统一使用 `ui/assets/combo-down.svg`，不得用 CSS 边框三角形或色块模拟箭头。
- 下拉框背景、边框、弹出列表、选中项颜色应与平台日志页面保持一致。
- 如页面需要额外尺寸约束，应通过控件 `minimumWidth` / `sizePolicy` 设置，不要复制一份独立 `QComboBox` 样式。

### 按钮

- 圆角
- 樱花粉或白粉半透明底
- 圆形按钮默认使用纯玫瑰色 `2px` 实边框
- 随聊天窗整体缩放按比例变化，最小不低于 `1px`
- hover 状态边框使用更深一档玫瑰色
- hover 时更亮、更暖
- 危险操作使用浅红色系

### SpinBox

- 不使用默认箭头
- 使用自定义 `RoseSpinBox`
- 右侧同列上下 `+ / -`
- 玫瑰红按钮

### 弹窗

- 必须与主题统一
- 确认框使用透明外层 + 内部卡片
- 非阻塞反馈使用轻量 toast
- 禁止出现圆角外露黑角

## 设置中心规范

### 导航栏

- 左侧导航从上到下固定为：
  - `🔑 API`
  - `👤 角色`
  - `🧠 记忆`
  - `🤖 Agent`
  - `🧩 插件`
  - `🔌 MCP`
  - `📝 对话日志`
  - `🧪 平台日志`
  - `⚙ 系统`
- `Agent` 保留机器人图标；`API` 不使用机器人图标，避免与 Agent 混淆。

### API 页面

- `保存 API 配置` 按钮出现在 API 页面
- 点击后必须二次确认
- 仅保存 API 配置
- 未保存编辑在切页后视为放弃
- TTS 语音合成里的参考语言和输出语言只保留下拉框本体，不再额外放置独立展开按钮。
- 参考语言和输出语言下拉框宽度应接近同组“引擎”控件，避免占满整行。
- ASR / STT 使用本地 faster-whisper 库：引擎选项为 `none` / `faster_whisper`，配置模型目录、设备、计算类型、语言、识别超时和录音参数；默认设备为 `cpu`、计算类型为 `int8`，避免无 CUDA 运行库环境误走 GPU；不得再展示 OpenAI API Key、OpenAI Base URL 或 faster-whisper 服务 URL。

### 系统页面

- 系统页面显示 `保存系统配置` 按钮，不再实时保存。
- 系统页保存只保存系统配置，不保存 API 配置。
- 保存后应立即应用到已打开聊天窗，包括字体、字号、聊天显示倍率、气泡参数和被动状态阈值。
- 外观设置不得继续挤在单个组里，应拆为基础外观、聊天显示、被动状态、被动气泡和网络。
- 字体必须使用系统字体下拉框，基于 `QFontDatabase.families()` 填充，并用每个字体自身的样式展示选项；控件允许编辑以保留未被 Qt 枚举到的字体名。

### 插件页面

- 操作要有即时反馈
- 只管理本地插件、内置插件权限和外部插件导入，不展示 MCP 服务器
- 内置插件标记为内置来源，不允许删除、覆盖或静默替换
- 外部插件导入必须校验 `plugin.py`，同名插件不得静默覆盖
- 外部插件搜索/下载只读取索引元数据，不自动执行远程代码；导入前必须提示第三方 Python 代码风险
- `system_control`、`web_automation` 的 `low / medium / high` 权限必须暴露给用户配置

### MCP 页面

- 操作要有即时反馈
- 只管理 MCP 服务器、连接诊断和 MCP 工具列表，不展示本地插件工具
- 支持刷新、新增、编辑、删除、启停
- 新增 / 编辑弹窗必须覆盖 `connect_timeout_seconds`、`request_timeout_seconds` 和 `retry_attempts`，避免用户只能改 YAML 才能治理慢连接或失败重试。
- `stdio` 服务器新增或编辑时必须提示会启动本地进程
- `sse` 服务器新增或编辑时必须提示会连接网络服务

### 角色页面

- 角色列表左侧显示迷你立绘
- 文件、立绘、同步操作都要有反馈
- 角色根目录核心文件 `prompt.md`、`soul.md`、`SKILL.md`、`sprites.yaml` 不允许从 UI 删除；用户可以编辑内容或通过备份恢复。

### 记忆页面

- 向量模型选择：下拉框优先自动扫描 `data/models/embedding/` 下含 `modules.json` 或 `sentence_bert_config.json` 的子目录，并兼容旧版 `data/models/` 直属向量模型目录。
- 语音识别模型选择：下拉框优先自动扫描 `data/models/stt/` 下含 `config.json` 和 `model.bin` 的 faster-whisper 模型目录，并兼容旧版 `data/models/` 直属 STT 模型目录；正斜杠 / 反斜杠、相对路径 / 可解析旧路径指向同一目录时必须去重。
- 模型下拉框条目右侧可显示 `×` 删除入口；点击后只从当前下拉列表移除该路径，不删除本地模型文件。
- 浏览按钮：支持选择外部模型路径，校验合法性后加入下拉框
- 删除按钮：仅对用户自定义路径可见，`data/models/` 扫描的条目不可删除
- 启用对话记忆：必须先选择向量模型才能勾选，勾选时二次确认
- 改动实时生效并写入 `memory.yaml`

### Agent 页面

- Agent 页面只管理规划、反思、多步推理和主动行为等 Agent 配置。
- 用户输入、角色回复、Thinking 预览、Planner / Tool / Reflector / MultiStep 等运行时间线已拆到 `对话日志` / `平台日志` 页面，不再放回 Agent 页面。
- Thinking 默认在日志工作台展示预览片段，避免日志页被长段推理文本刷满。
- 日志反馈应保持轻量、可连续阅读，不要做成阻塞式详情弹窗。

### 平台日志页面

- 不同 `source` 必须使用不同文字颜色，避免来源在列表和连续文本里无法区分。
- `STT` 必须是独立来源组，包含 `chat.stt` 与 `stt.faster_whisper`，并使用区别于 TTS / LLM 的专属来源颜色。
- 结构化列表选中项只改变背景，不改变来源文字颜色；选中态绘制时也必须把 `HighlightedText` 保持为来源色。
- 连续文本视图必须沿用来源配色，同时保留鼠标拖选、复制和全选能力。
- 自动刷新只在用户确实停留底部附近时跟随到底；用户翻阅历史记录时必须保持当前滚动位置，不得把离底部还有多条日志的位置视为底部。
- 页面不可见时必须暂停自动刷新，重新显示时再刷新当前视图，避免后台页面持续重建列表和详情区。
- 下方详情区在选中事件未变化时不得重写内容，避免阅读 JSON 详情时被弹回底部。
- 右键功能菜单、文本控件标准复制/粘贴菜单等所有 `QMenu` 实例必须使用 Sakura 浅色主题，不允许出现黑色背景。

## 聊天窗规范

- 无边框、透明背景
- 角色立绘占主体
- 底部毛玻璃对话面板
- 对话正文区域支持内部滚动，长文本不得把整体窗口撑出可用区域
- 对话面板外框默认使用纯玫瑰色 `3px` 实边
- 随聊天窗整体缩放按比例变化，最小不低于 `2px`
- 聊天窗整体缩放时，字体、输入框、按钮、滚动条和边框厚度应同步按比例变化
- 对话面板应保持较宽的横向占比，避免左右留白过多
- 对话面板高度应控制在窗口高度约 `45%`，避免压住立绘主体
- 角色立绘可与面板形成轻微前后遮挡关系，但不得影响文字可读性
- 支持拖拽、缩放、右键菜单
- 角色名和”我”颜色区分
- 文本行距舒适，不可拥挤
- 多段回复应保留段落语义，但不得使用浪费空间的整行空白分隔
- 主对话区域必须提供轻量运行状态条，覆盖思考、读屏、回复、语音合成、停止中、请求失败等状态。
- 运行状态条只承载当前轮次状态，不替代对话正文；失败时必须提供重试和打开平台日志入口。
- 忙碌态下发送按钮承担停止当前生成语义，图标 / tooltip / accessible name 必须同步变化。
- 发送按钮空闲态使用 `>` 文本符号，背景、hover、边框和圆角语言必须与语音按钮保持一致，不使用独立红色背景。
- 用户在忙碌态再次发送文本时，不应静默吞掉输入，应明确提示可先停止当前生成。
- 流式回复显示应按短间隔合帧刷新，避免每个 chunk 都触发完整富文本重绘；TTS 句段切分和入队仍需按原链路及时推进。
- 对话文本渲染应跳过同文本重复刷新，减少无意义富文本解析和滚动位置重置。
- 立绘加载应复用原图和近期目标尺寸缩放缓存，滚轮缩放和情绪切换不得重复读盘或反复执行同尺寸平滑缩放。

### 显示配置

- 聊天窗字体、输入框、按钮和边框应随 `SystemConfig.chat_display.font_scale` 统一缩放。
- `SystemConfig.chat_display.font_scale` 默认值为 `1.3`，保证首次启动时聊天字号比基础字号更易读。
- 被动气泡相关设置集中放在“被动气泡”分组；气泡缩放随 `SystemConfig.chat_display.bubble_scale` 调整宽度、内边距和圆角体感。
- 缩放只改变可读性和布局密度，不得改变主流程状态或消息内容。

### 被动互动气泡

- 被动互动不是系统设置开关，而是聊天窗运行态。
- 用户空闲达到 `passive_interaction.idle_threshold_seconds` 后自动进入被动状态。
- 右键菜单必须提供 `进入被动状态` / `退出被动状态`。
- 进入被动状态后主对话框应立即隐藏；只有主动状态才显示主对话框。
- 只有被动状态下的主动消息优先显示为独立气泡，不直接占用主对话框。
- 被动气泡自动隐藏后，如果聊天窗仍处于被动状态，主对话框也必须继续保持隐藏。
- 被动气泡的顶部应与主对话框上边缘对齐，避免贴近窗口底部。
- 被动气泡应靠近角色和聊天面板之间的视觉焦点，最大宽度受 `passive_interaction.bubble_max_width` 与窗口可用宽度共同限制；默认最大宽度为 `600px`。
- 被动气泡宽度应优先按当前字体下的文本单行宽度计算，只有文本宽度超过气泡最大宽度或窗口可用宽度时才换行；最小兜底宽度、最大宽度与窗口可用宽度都必须随气泡缩放和窗口缩放响应式变化。
- 被动气泡停留时长由 `passive_interaction.bubble_duration_seconds` 控制，到时自动隐藏。
- 拖拽窗口和滚轮缩放只刷新空闲计时，不应退出被动状态。
- 点击被动气泡应退出被动状态并恢复主对话框；右键菜单的 `退出被动状态` 是另一条显式退出路径。
- 用户开始输入、发送消息或启动语音输入时，应隐藏被动气泡，避免与主动对话状态冲突。
- 被动气泡仅承载短消息，不应展示工具结果、长回复、错误堆栈或需要用户确认的内容。
- 主动消息必须基于当前角色上下文生成；如角色定义了可用情绪，主动消息应带 `[emotion:情绪名]` 标签，聊天窗负责剥离标签并切换立绘。
- 主动消息必须进入普通聊天回复链路：记录平台日志、按现有规则切句、触发 TTS 合成和播放；被动状态只改变显示容器为气泡，不应跳过 TTS。

### STT 按钮状态

- ASR / STT 引擎为 `none` 时，麦克风按钮应禁用并提示语音输入未启用。
- 空闲可录音时，麦克风按钮表示“语音输入”，点击后进入录音状态。
- 语音输入未启用、当前 LLM 正在回复、当前 STT 正在识别或麦克风启动失败时，必须在聊天运行状态条给出即时反馈，并显示打开平台日志入口。
- 麦克风空闲态必须使用本地 SVG 图标，不依赖 emoji 字体；录音态可切换为文本停止符号，但按钮仍必须保持正圆。
- 麦克风按钮在聊天窗缩放、字体倍率变化、录音开始和录音结束后，都必须按当前按钮尺寸重新计算圆角，不得退回固定 `17px` 之类旧尺寸。
- 录音中应有明确状态提示，按钮再次点击应停止当前录音并进入后续转写流程；录音激活态图标必须稳定渲染，避免依赖当前字体不支持的 emoji 或显示成黑色方块。
- 转写中不得重复启动新录音；按钮应保持不可误触的忙碌状态。
- 转写成功后，识别文本进入输入框并复用发送主链路；转写为空或失败时，应恢复空闲状态并给出轻量反馈。
- STT 录音开始、录音结束、音频就绪、worker 启动、模型加载、转写完成、失败、超时和迟到结果忽略都应写入平台日志，便于实机调试。
- STT 转写不得让输入框无限停留在“正在识别...”；达到 `asr.transcribe_timeout_seconds` 后应释放当前识别状态、提示用户查看平台日志，并保留后台 worker 到自然结束后清理。
- 关闭聊天窗时必须取消录音并短等待转写 worker 收口，避免关闭态回调继续发送消息。
- 关闭聊天窗时必须等待 LLM、TTS、翻译、STT 和参考预热线程收口；不得在 `QThread` 仍运行时释放最后引用。

## 反馈交互规范

### 成功 / 失败反馈

- 默认使用非模态 toast
- 短消息、轻量、不打断操作
- toast 宽度应按当前字体下的文本单行宽度计算，只有超过最大宽度或宿主窗口可用宽度时才换行，避免一句短反馈被挤成多行。

### 高风险操作

- 删除、覆盖、保存确认等使用确认框
- 必须明确提示对象名称

## 禁止项

- 原生丑陋系统弹窗风格直接暴露
- 深色主题混入当前 sakura 风格
- 黑色 focus outline
- 控件圆角但窗口背景不透明导致黑角
- 保存逻辑和页面职责不一致
