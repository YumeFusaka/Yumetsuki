# Yumetsuki UI 规范

## 设计方向

Yumetsuki 的 UI 不是通用后台，也不是网页表单工具。它的风格应服务于：

- 桌宠角色感
- 粉白樱花主题
- 轻盈、柔和、现代
- 细节精致，但不过度装饰

## 颜色

- 主背景渐变：
  `#fff5f7 -> #ffebf2 -> #fae4f0`
- 强调色：
  `#d4567a`
- 深强调色：
  `#9b3060`
- 主文字：
  `#4a3040`
- 次级文字：
  `#6b4a5a`

## 通用控件风格

### 输入框

- 半透明浅白底
- 默认使用纯玫瑰色 `2px` 实边框
- 随聊天窗整体缩放按比例变化，最小不低于 `1px`
- focus 状态使用更深一档玫瑰色

### 按钮

- 圆角
- 樱花粉或白粉半透明底
- 圆形按钮默认使用纯玫瑰色 `2px` 实边框
- 随聊天窗整体缩放按比例变化，最小不低于 `1px`
- hover 状态边框使用更深一档玫瑰色
- hover 时更亮、更暖
- 危险操作使用浅红色系

### SpinBox

- 不使用默认箭头
- 使用自定义 `RoseSpinBox`
- 右侧同列上下 `+ / -`
- 玫瑰红按钮

### 弹窗

- 必须与主题统一
- 确认框使用透明外层 + 内部卡片
- 非阻塞反馈使用轻量 toast
- 禁止出现圆角外露黑角

## 设置中心规范

### API 页面

- `保存配置` 按钮只出现在 API 页面
- 点击后必须二次确认
- 仅保存 API 配置
- 未保存编辑在切页后视为放弃

### 系统页面

- 不需要单独保存按钮
- 改动实时生效并实时写入配置

### 插件 / MCP 页面

- 操作要有即时反馈
- 工具目录要有来源区分：
  - 本地插件
  - MCP
- 支持刷新、导入、删除、启停

### 角色管理页面

- 角色列表左侧显示迷你立绘
- 文件、立绘、同步操作都要有反馈

### 记忆页面

- 向量模型选择：下拉框自动扫描 `data/models/` 下含 `config.json` 或 `modules.json` 的子目录
- 浏览按钮：支持选择外部模型路径，校验合法性后加入下拉框
- 删除按钮：仅对用户自定义路径可见，`data/models/` 扫描的条目不可删除
- 启用对话记忆：必须先选择向量模型才能勾选，勾选时二次确认
- 改动实时生效并写入 `memory.yaml`

### Agent 页面

- 日志 Tab 显示混合时间线：
  - 用户输入
  - 角色回复
  - Thinking 预览
  - Planner / Tool / Reflector / MultiStep 等内部事件
- Thinking 默认显示预览片段，避免日志页被长段推理文本刷满
- 日志反馈应保持轻量、可连续阅读，不要做成阻塞式详情弹窗

## 聊天窗规范

- 无边框、透明背景
- 角色立绘占主体
- 底部毛玻璃对话面板
- 对话正文区域支持内部滚动，长文本不得把整体窗口撑出可用区域
- 对话面板外框默认使用纯玫瑰色 `3px` 实边
- 随聊天窗整体缩放按比例变化，最小不低于 `2px`
- 聊天窗整体缩放时，字体、输入框、按钮、滚动条和边框厚度应同步按比例变化
- 对话面板应保持较宽的横向占比，避免左右留白过多
- 对话面板高度应控制在窗口高度约 `45%`，避免压住立绘主体
- 角色立绘可与面板形成轻微前后遮挡关系，但不得影响文字可读性
- 支持拖拽、缩放、右键菜单
- 角色名和”我”颜色区分
- 文本行距舒适，不可拥挤
- 多段回复应保留段落语义，但不得使用浪费空间的整行空白分隔

## 反馈交互规范

### 成功 / 失败反馈

- 默认使用非模态 toast
- 短消息、轻量、不打断操作

### 高风险操作

- 删除、覆盖、保存确认等使用确认框
- 必须明确提示对象名称

## 禁止项

- 原生丑陋系统弹窗风格直接暴露
- 深色主题混入当前 sakura 风格
- 黑色 focus outline
- 控件圆角但窗口背景不透明导致黑角
- 保存逻辑和页面职责不一致
