# Tauri UI 结构复刻规则

## 目标

本文定义 Tauri UI 重构期间的结构复刻硬规则。

Tauri / Vue 新界面必须以现有 PySide6 `ui/` 源码为硬性结构和功能参照。迁移目标是完整保留旧 UI 的入口、层级、页面结构、功能项、状态语义和运行链路；样式允许 Web 化调整，不要求像素级一致。

## 核心结论

- `ui/` 源码是迁移期唯一硬性 UI 结构来源，不是灵感参考。
- 每个 Vue 页面、窗口或组件设计和实现前，必须先读取对应的 `ui/` 旧实现文件。
- 如果找不到明确对应旧文件，必须暂停并说明缺口，不能自由设计替代结构。
- 截图只作为辅助参考，不作为主验收依据。
- 包管理器使用 `pnpm`；前端测试和脚本命令不得改用 npm / yarn。

## 必须保留的内容

迁移时必须完整保留以下内容：

- 页面入口：旧 UI 中可进入的窗口、页面、弹窗、菜单和操作入口，新 UI 必须有同等入口。
- 信息架构：设置中心导航顺序、页面层级、分组归属、字段归属和主要操作归属不得随意重排。
- 功能项：旧 UI 中已有的配置项、按钮、菜单项、右键菜单、弹窗、导入 / 导出、测试连接和诊断入口不得缺失。
- 状态语义：`loading`、`empty`、`dirty`、`saving`、`error`、`disabled`、`busy`、`streaming`、`recording`、`retryable` 等状态必须有对应表达。
- 行为语义：保存、取消、失败回滚、保留草稿、确认高风险操作、停止当前任务、关闭清理和迟到结果丢弃必须与旧 UI 等价。
- 运行链路：聊天、TTS、STT、OCR、日志刷新、插件、MCP、诊断和配置广播等链路不能只复刻表面控件，必须保留原有生命周期和边界条件。

## 允许差异

以下差异允许存在：

- CSS 颜色、间距、圆角、阴影、字体细节和 Web 控件外观。
- Vue 组件拆分、Pinia store 分层和 Tauri command / event 的实现结构。
- 为适配 Web、Tauri capability、响应式布局和无障碍要求而进行的局部视觉调整。
- 把 PySide6 特有控件替换为等价 Web 控件，但不得减少可见功能项或改变操作归属。

以下差异不允许存在：

- 因重新设计导致页面入口、字段、按钮、菜单、弹窗或状态反馈缺失。
- 未说明理由就改变设置中心导航顺序、页面分组顺序或字段归属。
- 把旧 UI 中的失败回滚、保留草稿、取消语义、线程 / 任务清理等隐含行为省略。
- 脱离 `ui/` 源码和现有文档自由设计新页面。

## 迁移前对照流程

每个页面或组件迁移前必须完成以下流程：

1. 找到旧 UI 对照文件。
2. 读取旧实现和相邻框架文件。
3. 写出旧 UI 对照摘要。
4. 再开始 Vue / Tauri 设计或实现。
5. 完成后用结构 parity 清单验证。

旧 UI 对照摘要至少包含：

```text
旧 UI 对照文件：
- ui/settings/window.py
- ui/settings/pages/system_page.py

结构来源：
- 设置中心左侧导航和右侧页面框架
- 系统页分组顺序
- 保存按钮归属系统页
- 保存失败时回滚内存配置、页面草稿和已打开聊天窗配置

必须保留：
- 页面入口
- 分组和控件顺序
- 保存 / 失败反馈 / 回滚语义
- 与聊天窗运行态的配置广播

允许差异：
- CSS 视觉、控件外观和响应式适配
```

没有对照摘要的 Tauri UI 设计或实现视为不合格。

## 结构 parity 清单

每个迁移项完成时必须逐项确认：

- [ ] 已读取对应 `ui/` 旧实现文件。
- [ ] 已读取必要的父级窗口、主题、通用控件或模板文件。
- [ ] 页面入口保留。
- [ ] 页面层级和分组顺序保留。
- [ ] 控件项、字段项和操作项无缺失。
- [ ] 按钮、菜单、右键菜单、弹窗和反馈入口无缺失。
- [ ] 加载、空态、修改未保存、保存中、错误、禁用和忙碌状态有对应表达。
- [ ] 保存、取消、失败回滚、保留草稿和高风险确认语义保留。
- [ ] 关闭、取消、任务清理、迟到结果丢弃和资源释放语义保留。
- [ ] 已补充或更新对应测试；如无法自动化，已说明人工验证点和剩余风险。

## 常用对照入口

迁移时优先从以下旧文件开始阅读：

- 设置中心框架：`ui/settings/window.py`
- 设置页面：`ui/settings/pages/*.py`
- 聊天窗：`ui/chat/window.py`
- 聊天模板：`ui/chat/templates/*`
- TTS / STT / 播放边界：`ui/chat/tts_pipeline.py`、`ui/chat/stt_recorder.py`、`ui/chat/audio_backends.py`
- 立绘：`ui/chat/sprite.py`
- 启动加载窗：`ui/startup/loading_window.py`
- 主题与通用控件：`ui/theme.py`、`ui/widgets/*`、`ui/text_metrics.py`
- 图标和资源：`ui/assets/*`

## 截图使用规则

截图可以作为人工辅助材料，但不要求完整覆盖所有状态。

- 有截图时，新 UI 可参考截图确认大结构和入口位置。
- 没有截图时，仍以 `ui/` 源码和结构 parity 清单为准。
- 截图不得替代源码审计，因为隐藏状态、失败回滚、异步任务、关闭清理和权限确认通常只在代码中体现。

## 交付要求

每个 Tauri UI 迁移任务的最终说明必须包含：

- 读取过的旧 UI 对照文件。
- 保留的结构和功能点。
- 允许差异的范围。
- 已运行的验证命令；前端命令必须使用 `pnpm`。
- 未能自动化验证的剩余风险。
