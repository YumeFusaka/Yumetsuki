# 聊天窗口布局与排版优化设计

> 日期：2026-05-22

## 概述

本次优化聚焦桌宠聊天窗口的空间利用和主题层次感，不改动聊天主流程、Agent、LLM、角色资源加载和设置中心行为，仅在 `ui/chat/window.py` 内做定向调整。

目标如下：

1. 底部对话面板体感宽度明显增加，同时高度缩短约 10%
2. 角色立绘整体下移，形成“站在面板后方，底部略低于面板上缘”的视觉关系
3. 毛玻璃对话面板透明度降低一档，让立绘透出更多
4. 对话框、输入框、按钮边缘增加更贴合樱花主题的层次化描边
5. 压缩 LLM 多段回复中的多余空白，减少段与段之间浪费空间的问题

## 设计原则

- 保持现有桌宠聊天窗口结构，不更换为更重的富文本控件
- 延续现有 sakura 主题，不引入偏离文档规范的新色系
- 优先用温和参数调整解决问题，避免把窗口横向拉得过胖
- 文本仍保留分段语义，但显示层不再保留多余空白行
- 缩放行为必须继续成立，避免仅在 1.0 倍下好看

## 方案选择

本次采用“保守增强”方案，即在当前 `QLabel + QScrollArea + GlassPanel` 结构上进行局部调参和样式增强。

不采用 `QTextEdit/QTextBrowser` 重做文本区，原因如下：

- 这次问题集中在空间浪费和视觉层次，不需要引入更重的文本控件
- 保留现有结构可以降低回归风险
- 对当前需求而言，显示层规范化和样式微调已经足够

## 一、布局调整

### 1. 整体宽度策略

不采用“只把面板硬拉宽”的做法，而是通过两类调整叠加实现约 30% 的体感增宽：

- 聊天窗口基准宽度小幅增加
- 底部面板左右外边距缩小，让面板更贴近窗口边缘

这样既能显著提升对话区可用宽度，也不会让整体造型失衡。

### 2. 对话面板比例

- 面板高度在当前基础上缩短约 10%
- 面板仍固定在底部区域，但保持圆角毛玻璃卡片样式
- 面板内部上下留白同步微调，避免变矮后显得拥挤

### 3. 立绘落点

- 立绘区域的可用高度和重载目标尺寸会重新计算
- 立绘整体显示位置下移
- 目标效果是立绘底部略低于对话面板上缘，形成轻微前后遮挡感

不改变立绘加载逻辑，也不修改角色资源本身。

## 二、视觉样式调整

### 1. 面板透明度

- `GlassPanel` 的填充透明度降低一档
- 保留文字可读性，不让面板变成过于轻薄的漂浮信息层

### 2. 层次化描边

对以下部位统一增加更有层次的主题描边：

- 毛玻璃面板外框
- 输入框边框
- 圆形按钮边框

描边风格采用樱花主题的浅深两级关系：

- 常态：柔和浅粉主边
- hover/focus：更清晰的玫瑰强调边

目标不是高对比发光效果，而是“更精致、更有层次”的边缘感。

### 3. 细节一致性

- 输入框、按钮、滚动条颜色继续服从现有 sakura 主题
- 缩放后描边厚度、圆角、padding、按钮尺寸继续等比变化

## 三、文本排版优化

### 1. 空白压缩策略

对 LLM 输出仅做显示层规范化，不修改原始对话语义：

- 去除首尾无意义空白
- 把连续 3 个及以上换行压缩为 2 个换行
- 渲染时不再制造“额外空一整行”的段间效果

这样可以保留多段结构，但不再浪费对话面板高度。

### 2. 行高与段落占用

- 当前正文 `line-height` 小幅下调
- 段落之间不再依赖大空白做区分
- 每段之间只保留克制的视觉分隔

目标是让长回复仍然易读，但单位高度容纳更多内容。

### 3. 兼容性

该处理仅影响聊天窗显示，不影响：

- Agent 日志
- LLM 内部消息历史
- 记忆系统写入内容

## 四、涉及代码点

仅修改 `ui/chat/window.py`，主要落点如下：

1. `GlassPanel.paintEvent()`
   - 调整填充透明度
   - 升级描边层次

2. `ChatWindow._apply_scale()`
   - 调整基准尺寸
   - 重算底部面板几何
   - 调整立绘区域与立绘落点

3. `ChatWindow._rebuild_stylesheet()`
   - 更新输入框、按钮、滚动条和面板内部间距
   - 加入更有层次的主题边缘样式

4. `ChatWindow._set_dialog_text()`
   - 对显示文本做换行规范化
   - 下调行高
   - 保留分段但压缩空白

## 五、验证方式

### 手动验证

运行 `python main.py`，重点确认：

1. 面板明显比当前更宽，且高度略矮
2. 立绘底部位置下移，视觉上略低于面板上缘
3. 面板、输入框、按钮边缘更有主题层次
4. 多段回复不再出现浪费空间的整行空白
5. 长文本仍可滚动查看
6. 滚轮缩放和右键缩放后，样式比例依然协调

### 回归关注点

- 不影响消息发送与流式刷新
- 不影响角色表情切换
- 不出现圆角黑边或不透明背景露出
- 不因为文本规范化导致段落完全粘连

## 六、范围边界

本次不包含以下内容：

- 聊天控件从 `QLabel` 改为 `QTextEdit/QTextBrowser`
- 调整 Agent 日志页面
- 修改 LLM 输出逻辑或提示词
- 修改角色立绘资源、裁切方式或角色包结构
